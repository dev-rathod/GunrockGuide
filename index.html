01 — About the Project

What is
GunrockGuide?

GunrockGuide is an embedded systems navigation tool built to solve a real problem every UC Davis student faces — getting lost in the sprawling academic buildings on campus.

Using a combination of sensors, microcontrollers, and wireless communication, GunrockGuide provides step-by-step directional guidance to help you find any room on campus without pulling out your phone.

Built as the EEC 172 final project, every component — from hardware design to cloud integration — was engineered from scratch by Amal and Dev.

📍

Real-Time Navigation

Turn-by-turn directions to any room number on campus using embedded sensor data.

📡

Wireless Connectivity

Communicates over Wi-Fi to fetch map data and user destinations on demand.

🖥️

Embedded Display

Compact OLED display shows directions and current location at a glance.

☁️

Cloud Backend

AWS powers the routing logic and stores building floor plans for instant lookup.

03 — Development Story

Our
Journey

Phase 1

Problem Discovery

We both experienced the frustration of being late to class because buildings like Kemper and Bainer are impossible to navigate. We decided to solve it with embedded systems.

Phase 2

Hardware Design

Selected the CC3200 microcontroller for its built-in Wi-Fi capabilities. Designed the circuit schematic, chose sensors, and prototyped the OLED display interface.

Phase 3

Software & Routing

Implemented pathfinding algorithms in C on the embedded device and built the AWS Lambda backend to serve building map data as JSON over HTTPS.

Phase 4

Integration & Testing

Brought hardware and software together. Ran live navigation tests inside Kemper Hall — successfully guided us from the entrance to Room 1150 on the first try.

Phase 5

Final Demo

Polished the user interface, documented the system, and presented GunrockGuide as our EEC 172 final project. Future work: expand to more buildings and add voice output.

04 — Cloud Infrastructure

AWS
Integration

GunrockGuide's cloud backend is built on Amazon Web Services, enabling real-time data delivery to the embedded device over a lightweight REST API.

λ

AWS Lambda

Serverless functions handle routing requests from the device. When a room number is entered, Lambda computes the path and returns step-by-step directions as JSON.

Serverless Functions
🗄

Amazon DynamoDB

Building maps and room-to-node mappings are stored in DynamoDB. Fast key-value lookups ensure the embedded device gets its response in under 300ms.

NoSQL Database
🔗

API Gateway

Exposes a secure HTTPS endpoint that the CC3200 calls over Wi-Fi. Handles authentication, rate limiting, and routes requests to the appropriate Lambda function.

REST API
🪣

Amazon S3

Floor plan images and static assets for the web interface are hosted in S3 buckets with CloudFront CDN delivery for low-latency access from anywhere on campus.

Object Storage
